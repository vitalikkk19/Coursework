	\subsection{Класифікація атак на стегоситеми}
	\begin{frame}
	Стегосистему вважають \textbf{зламаною}, якщо зловмисник зміг довести, що в перехопленому контейнері наявна прихована інформація. Якщо йому це не вдалось, то кажуть, що система \textbf{стійка}.
	Види атак на стегосистему:
	\begin{itemize}
		\item атака, що грунтується на знанні заповненого контейнера;
		\item атака, що грунтується на відомому повідомленні ;
		\item атака, що грунтується на прихованих даних чи заповнених контейнерів, що обирались зловмисником;
		\item атака, що грунтується на відомому пустому контейнері ;
		\item атака, що грунтується на виборі порожнього контейнера зловмисником;
		\item атака, що грунтується на знанні математичної моделі контейнера.
	\end{itemize}
	Етапи зламування стегосистеми:
	\begin{itemize}
		\item розкриття факту існування прихованої інформації;
		\item добування прихованих даних;
		\item перетворення секретних даних;
		\item заборона поширення скритого повідомлення.
	\end{itemize}
	\end{frame}

	\subsection{Метод заміни НЗБ в BMP форматі}
	\begin{frame}
	Загальний алгоритм приховування даних в BMP форматі зображення:
	\begin{itemize}
		\item вікриття файлу контейнера для прихованих даних;
		\item вибір кольору палітри контейнера (червона, синя, зелена);
		\item  відкриття файлу з даними, які треба приховати;
		\item конвертування даних в двійковий формат для засекречення в масиви по 8 біт (всі характеристики зображення та інтенсивність кольорів пікселів); 
		\item заміна найменш значущого біта пікселя контейнера бітом прихованих даних відповідно до величини ключа;
		\item зберігання заповненого контейнера, тобто зміненого файлу, у форматі BMP відповідно до правил зберігання.
	\end{itemize}
	Для видобування даних потрібно:
	\begin{itemize}
	\item знати ключ, з яким в контейнері розміщувались дані;
	\item відкрити файл з секретною інформацію, щоб отримати значення кольорів пікселів повного контейнера;
	\item використовуючи ключ, вибрати найменш значущі біти кольорів пікселів та згенерувати з них секретні дані.
	\end{itemize}
	\end{frame}

	 \subsection{Методом \mbox{Коха–Жао} в JPEG форматі}
	\begin{frame}
	 \emph{Метод Коха-Жао можна описати наступним алгоритмом:} \\
	 \textbf{Крок 1}: Спочатку початкове зображення розділюють на блоки розміром 8$\times$8 пікселів, до яких застосовують дискретно косинусні перетворення, причому в одному блоці приховують один біт даних.\\
	 \textbf{Крок 2}:
	 Застосувати дискретно-косинусне перетворення до кожного блоку. В результаті отримаємо набір матриць коефіцієнтів $D_i$ розмірності 8. \\
	 \textbf{Крок 3}:
	 Обираємо блоки для вбудовування та записуємо в кожен блок по одному біту даних. \\
	 \textbf{Крок 4}:
	 У всіх блоках потрібно вибрати два коефіцієнти перетворення, які симетричні головній діагоналі.\\
	\end{frame}

	\begin{frame}
	\textbf{Крок 5}:
		Щоб передати біт <<0>>, необхідно, щоб різниця абсолютних значень
		коефіцієнтів Д-К перетворень була більша ніж якась порогова величина. Щоб \mbox{передати} біт <<1>>, необхідно, щоб різниця була меншою ніж якась порогова величинаю. Тому, наприклад, для вбудовування нульового біту збільшуть перший, але зменшують другий коефіцієнт на однакову величину та навпаки. \\
	\textbf{Крок 6}: Четвертий та 5 пункт потрібно виконати для всіх блоків \\
	\textbf{Крок 7}: Виконати обернене дискретно косинусне перетворння для всіх блоків.\\
	\emph{Для розшифровування інформації:}\\
	\textbf{Крок 4}: Потрібно виділити матриці колірних компонент зображення\\
	\textbf{Крок 5}: Виконуємо розбиття матриці на блоки $Ic$\\
	\textbf{Крок 6}:  До кожного блоку застосовуємо дискретно-косинусне
	перетворення. \\
	\end{frame}

	\begin{frame}
	\begin{itemize}
		\item Стеганографія в поєднанні з криптографією, є потужним захистом інформації.
		\item Метою стеганографічних алгоритмів є забезпечити  приховування факту наявності даних, що потребує захисту.
		\item Надійним алгоритмом захисту даних в зображеннях є алгорит маскування, що використовують видимі аспекти зображення, ніж, скажімо, модифікація модетів заміни найменш значущого біта щодо різних видів обробки зображень.
		\item Найбільш надійним для захисту даних в текстових форматах є метод знаків одного накреслення.
		\item Для аудіо-форматів найефективнішим вважають метод заміни найменш значущого біта. 
		\item Метод ЗНЗБ для приховування даних в BMP форматі є простим та може приховувати великий об'єм даних.
		\item Збільшуючи розмір вбудованих даних зменшується якість зображення.
		\item Метод Коха-Жао є стійким до більшості відомих стегоатак, але, має низьку пропускну здатність
	\end{itemize}
